## 分頁二：合約轉換

### 轉換邏輯

```
對業主合約 (.docx)
  甲方 = 業主
  乙方 = 志昌 / 瀚昱 / 毅源
         ↓
  1. 偵測乙方公司名稱（志昌／瀚昱／毅源）
  2. 更新合約前文甲乙方稱謂
       甲方：業主 → 志昌/瀚昱/毅源
       乙方：志昌/瀚昱/毅源 → 悠勢科技
  3. 填寫「甲方指定匯款銀行帳戶」（銀行名稱、帳戶名稱、帳號）
  4. 替換簽名頁（立契約書人）
  5. 強制所得代號為「法人 00 發票」
  6. XML 格式驗證
  7. 附加銀行帳戶影本圖片（最後一頁，僅瀚昱／毅源）
         ↓
對悠勢合約 (.docx)
  甲方 = 志昌 / 瀚昱 / 毅源
  乙方 = 悠勢科技股份有限公司
```

> 合約條款本文完全不動，只替換甲乙方資訊與銀行帳戶。

### 銀行帳戶影本圖片

瀚昱、毅源的轉換結果最後一頁會自動附上銀行帳戶影本（存放於 `pic/` 資料夾）：

| 公司 | 圖片檔名 |
|------|----------|
| 瀚昱開發股份有限公司 | 瀚昱-凱基城東.jpg |
| 毅源開發股份有限公司 | 毅源-凱基城東.jpg |

> 志昌不附圖（使用國泰世華，帳戶資料直接填入合約內文即可）。

### 輸出檔名規則

```
原始檔名(對悠勢合約).docx

範例：新豐--停車位租賃契約書(對業主合約).docx
  → 新豐--停車位租賃契約書(對業主合約)(對悠勢合約).docx
```

### 注意事項
- 僅支援 `.docx` 格式，不支援 PDF
- 系統自動識別合約中的公司名稱（志昌資產管理股份有限公司 / 瀚昱開發股份有限公司 / 毅源開發股份有限公司）
- 若合約中找不到上述公司名稱，會回傳錯誤提示
- 支援分潤版（段落帳戶格式）與租金版（表格帳戶格式）
- 上傳的檔案僅在記憶體中處理，不儲存於伺服器

## 檔案結構

```
contract-generator/
├── main.py                  # FastAPI 後端（路由 + Google Sheets 記錄）
├── converter.py             # 合約轉換邏輯（XML 操作 + 圖片附加）
├── generator.py             # 合約產生邏輯（docxtpl 範本渲染）
├── requirements.txt
├── render.yaml              # Render 部署設定
├── static/
│   └── logo.png
├── templates/
│   └── index.html           # 前端介面（產生 + 轉換兩分頁）
├── pic/                     # 銀行帳戶影本圖片（瀚昱、毅源）
│   ├── 瀚昱-凱基城東.jpg
│   └── 毅源-凱基城東.jpg
└── 自動產生合約範本/
    ├── template_rent.docx
    ├── template_profit.docx
    ├── template_rent_uspace.docx
    └── template_profit_uspace.docx
```

## 注意事項

| 項目 | 說明 |
|------|------|
| 輸入格式 | 僅支援 `.docx`，不支援 PDF |
| 必填欄位 | 詳見上方必填欄位清單 |
| 非必填空白 | 未填的非必填欄位會自動留底線空白，供手寫補填 |
| 所得代號 | 產生器：依勾選填入合約與表頭；轉換器：固定為法人 00 發票 |
| 日期格式 | 系統自動將西元年轉為民國年（顯示於合約內） |
| 乙方% 自動計算 | 分潤型填入甲方%後，乙方%自動補為 100 - 甲方% |
| 銀行帳戶影本 | 瀚昱、毅源轉換後最後一頁自動附上銀行帳戶影本（JPEG） |
| 免費方案限制 | 閒置 30 分鐘後休眠，第一次開啟需等 30～50 秒喚醒 |
| 資料安全 | 合約檔案僅在記憶體中處理，不儲存於伺服器 |
| 使用量記錄 | 每次使用自動寫入 Google Sheets，背景執行不影響速度 |
